# 🛡️ サイバーセキュリティ・ダイジェスト 2026-07-08

2026年7月8日に取得されたソースに基づき、サイバーセキュリティニュースを分析します。

## 🔴 重大インシデント
*   **米国地方自治体による100万ドルの身代金支払い**
    米国オハイオ州のユニオン郡と思われる地方自治体が、サイバー脅迫グループ「Kairos」に対し100万ドルの身代金を支払ったと報じられました [1-3]。2025年5月の侵入で約2TB（160万ファイル）のデータが盗まれ、45,487人の個人情報が含まれていました [2, 3]。当局はデータの公開を阻止するため、3週間の交渉の末にビットコインで支払いを行いました [2, 4]。
*   **Fortinet製品の認証情報大規模漏えい（FortiBleed）**
    「FortiBleed」と呼ばれる事案により、194カ国にわたる86,644台以上のFortinet機器のログイン情報が漏えいした可能性が指摘されています [5]。JPCERT/CCは、この漏えいデータに**国内企業の関連情報が含まれていることを確認**しています [6]。攻撃者は窃取した情報を悪用してVPN経由で内部ネットワークへ侵入し、Windowsドメイン認証情報をさらに解読するツールを保有している事例も確認されています [6]。

## 🟠 脆弱性情報
*   **Gitea における認証バイパスの脆弱性**
    *   **CVE番号 / CVSSスコア:** CVE-2026-20896 / **9.8** [7]
    *   **概要:** 公式Dockerイメージのデフォルト設定の不備により、単一のHTTPヘッダーに有効なユーザー名を含めるだけで、パスワードなしで管理権限を含むリポジトリやシークレットにアクセス可能です [7-9]。
    *   **推奨対応:** 対策が導入されたバージョン **1.26.3 または 1.26.4 以降**へ早急にアップデートしてください [10, 11]。
*   **Microsoft Exchange Server のなりすましの脆弱性**
    *   **CVE番号:** CVE-2026-42897 [12]
    *   **概要:** リモートから任意のコードを実行されるなどの可能性がある脆弱性で、すでに悪用が確認されています [12]。
    *   **推奨対応:** Microsoft Update等を通じて、2026年6月のセキュリティ更新プログラムを適用してください [13]。
*   **Adobe AcrobatおよびReaderの脆弱性**
    *   **アドバイザリ番号:** APSB26-63 [14]
    *   **概要:** 脆弱性を悪用したコンテンツを開いた場合、任意のコードが実行される可能性があります [14]。
    *   **推奨対応:** メニューの「ヘルプ」から「アップデートの有無をチェック」を選択し、最新バージョン（26.001.21662等）に更新してください [15]。

## 🟡 攻撃キャンペーン
*   **DEBULL ツールによる Microsoft 365 アカウントの奪取**
    OAuth 2.0 の「デバイス認証フロー」を悪用し、多要素認証（MFA）を回避してアカウントを乗っ取るフィッシングキャンペーンが確認されています [16, 17]。攻撃者は「DEBULL」と呼ばれる再利用可能なツール層を使用し、Teams風のルアー（誘い文句）でユーザーに正規のログイン画面で攻撃者用のコードを入力させ、セッショントークンを盗み出します [16, 18, 19]。
*   **RedWing Android 銀行詐欺サービス (MaaS)**
    Telegram上で「RedWing」というAndroidマルウェアのレンタルサービスが展開されています [20]。フィッシングリンクを通じて偽のアプリストアから不正アプリをインストールさせ、アクセシビリティサービスを悪用して銀行のログイン情報やワンタイムパスワードを窃取します [21, 22]。このサービスは動画ガイド付きで販売されており、スキルの低い犯罪者でも利用可能です [21]。

## 🤖 AIセキュリティ
*   **CISAによるAI監査の実施:** 米国サイバーセキュリティ・インフラセキュリティ局（CISA）は、AnthropicのAIモデル「Mythos」を使用して連邦政府のソフトウェアコードをスキャンしており、すでに**「多数」の脆弱性を発見**したと報じられています [23-25]。
*   **Google Dialogflow CX の脆弱性「Rogue Agent」:** 共有ランタイム環境の不備を突くことで、一つのエージェント編集権限を持つ攻撃者が、同一プロジェクト内の他者のチャットボットを乗っ取ることが可能でした [26, 27]。これにより、会話の窃取や、ボットを通じたフィッシングメッセージの送信が行えるリスクがありました [26, 28]。
*   **AI搭載型フィッシング基盤:** 「EvilTokens」などの攻撃プラットフォームは、AIを使用して大量の窃取メールから金融関連のやり取りを自動特定し、ビジネスメール詐欺（BEC）の文面を作成する機能を備えています [29]。

## 🔵 業界動向
*   **AI活用による脆弱性発見の自動化:** 政府機関がAIを積極的に導入し、公開前や運用中のコードから脆弱性を高速に特定する手法が一般化しつつあります [24, 25]。
*   **フィッシング・アズ・ア・サービス (PhaaS) の高度化:** デバイス認証フローの悪用や、トークンの永続化、AIによる自動化機能を一式揃えた「ARToken」などの攻撃パネルが流通しています [29, 30]。
*   **Androidにおける「オンデバイス不正」へのシフト:** パスワードを盗むだけでなく、アクセシビリティサービス等を悪用して被害者の正規セッション内で直接操作を行う手口が増加しています [31]。
*   **AIガードレールを巡る対立:** 兵器利用制限などの安全策解除を巡り、米政府がAI企業（Anthropic）を供給網リスクとして指定するなど、安全保障と技術発展の緊張が高まっています [32, 33]。
