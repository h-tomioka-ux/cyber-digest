# 🛡️ サイバーセキュリティ・ダイジェスト 2026-07-06

> ソース: RSS 6フィード (12記事) ／ NotebookLM分析

2026年7月6日に取得されたサイバーセキュリティニュースに基づき、情報を分析して報告します。

## 🔴 重大インシデント
*   **Medtronicにおける380万人規模のデータ漏洩**
    医療技術大手のMedtronicは、2026年4月に恐喝グループ「ShinyHunters」によってITシステムが侵害され、約383万人の患者の個人情報や健康情報が窃取されたことを確認しました [1-3]。流出したデータには氏名、連絡先、社会保障番号が含まれており、同社は対象者に対し24ヶ月間の無料クレジット監視サービスを提供しています [3, 4]。
*   **米国政府機関に対する100万ドルのデータ窃盗恐喝事件**
    オハイオ州ユニオン郡と推定される政府機関が、攻撃グループ「Kairos」に対し、窃取されたファイルの公開を阻止するために約100万ドル（約9.44 BTC）を支払いました [5-7]。この事件ではファイルの暗号化は行われておらず、盗み出した機密データそのものを人質にする「純粋なデータ窃盗恐喝」の手口が用いられています [5, 8]。
*   **KDDIによる1422万人分のデータ漏洩公表**
    日本の通信事業者KDDIが、BIGLOBEやJCOMなど5つのISPに関連する利用者約1422万人分のメールアドレスとパスワードが漏洩した可能性があると発表しました [9]。

## 🟠 脆弱性情報
*   **FatFsライブラリの未パッチの脆弱性 (CVE-2026-6682など)**
    *   **CVSSスコア:** 7.6 (High) [10]
    *   **概要:** 数百万台の埋め込みデバイス（監視カメラ、ドローン、ATM等）で使用されているファイルシステムライブラリ「FatFs」に、メモリ破損やコード実行を可能にする7つの脆弱性が発見されました [10-12]。
    *   **推奨対応:** 開発者からの公式パッチが提供されていないため、物理ポートへのアクセス制限を徹底し、各デバイスベンダーから提供されるファームウェアアップデートを監視してください [13, 14]。
*   **Langflowの認証欠如の脆弱性 (CVE-2025-3248)**
    *   **CVSSスコア:** 9.8 (Critical) [15]
    *   **概要:** AIアプリ構築フレームワークのLangflowにおいて、認証なしで任意のPythonコードを実行可能な欠陥が悪用され、後述するAIによる自律的攻撃に利用されました [15, 16]。
    *   **推奨対応:** 露出したアプリケーションサーバーのセキュリティ硬化を実施し、CISAの注意喚起に従って最新のパッチを適用してください [16, 17]。
*   **Microsoft Exchange Server のなりすましの脆弱性 (CVE-2026-42897)**
    *   **CVSSスコア:** 未記載（ソース内）
    *   **概要:** 2026年6月の月例セキュリティ更新プログラムで修正された脆弱性で、既に悪用が確認されています [18]。
    *   **推奨対応:** Microsoft Update等を利用して、速やかにセキュリティ更新プログラムを適用してください [19]。

## 🟡 攻撃キャンペーン
*   **北朝鮮ハッカーによる「PolinRider」キャンペーン**
    北朝鮮の脅威アクター（Contagious Interview）が、npmやGoなどのパッケージレジストリに108個の悪意あるパッケージを公開しました [20, 21]。求人活動を装ってソフトウェア開発者に接触し、VS Codeのタスクファイル等を悪用してマルウェアを実行させるサプライチェーン攻撃を展開しています [22, 23]。
*   **Fortinet製品の認証情報漏洩「FortiBleed」**
    194カ国にわたる86,644台のFortiGate等の機器のログイン情報が漏洩し、ロシアの攻撃者らによって運用されています [24, 25]。過去のインシデントで流出した情報の再利用が主な原因ですが、設定ファイルのオフライン解析によって管理者パスワードが解読されるリスクが指摘されています [24, 26]。

## 🤖 AIセキュリティ
*   **エージェント型AIによる自律的ランサムウェア攻撃**: 脅威アクター「JadePuffer」が、LLMエージェントを利用してLangflow経由で侵入し、リアルタイムで行動を修正しながら偵察、横展開、データベースの暗号化を行う一連の攻撃を自動化しました [15, 27-29]。
*   **AIによるゼロデイ脆弱性の発見**: LLMファジングを用いたAIエージェントにより、FFmpegや7-Zipなどの主要なオープンソースプロジェクトから数十のゼロデイ脆弱性が発見されています [30, 31]。
*   **AIエージェントを狙った攻撃**: 毒された（Poisoned）ツール記述や隠されたコンテンツ注入により、自律型AIエージェントにデータを漏洩させたり、誤った行動をとらせたりする「AIエージェントの罠」が警告されています [32-34]。

## 🔵 業界動向
*   **暗号化なしの恐喝の増加**: ランサムウェア攻撃のうち暗号化を伴うものは約半分に減少し、データの窃盗そのものを交渉材料にする手口が過去6年間で最も一般的になっています [8]。
*   **AI生成コードの監査戦略**: AIによるコード生成が常態化する中、CISOには開発慣行やAIツールの使用、生成されたソフトウェアのリスクを監視する新しい監査戦略が求められています [35-37]。
*   **エージェント型AIのコスト管理**: セキュリティプラットフォームへのAIエージェント導入にあたり、検知性能の向上と、トークン消費やインフラ維持に伴うコスト増大のバランスが課題となっています [38-40]。
*   **サプライチェーン攻撃の巧妙化**: 北朝鮮のキャンペーンに見られるように、Gitの履歴書き換えや偽のフォントファイルによるコード難読化など、開発者のツールチェーンを標的にした隠蔽技術が高度化しています [41, 42]。
