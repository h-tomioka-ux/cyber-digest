# 🛡️ サイバーセキュリティ・ダイジェスト 2026-05-15

> ソース: RSS 6フィード (12記事) ／ NotebookLM分析

## 🔴 重大インシデント
**Cisco SD-WAN製品を標的とした高度な攻撃グループ「UAT-8616」によるゼロデイ攻撃**
高度な攻撃グループ「UAT-8616」が、Cisco Catalyst SD-WANの脆弱性（CVE-2026-20182など）を悪用して管理権限を取得し、NETCONF設定の変更やroot権限への昇格を試みています [1-3]。この活動では10個の異なる活動クラスターが確認されており、Webシェル（Godzilla、Behinder等）の設置や、仮想通貨マイナー、認証情報窃取ツールの展開が行われています [4, 5]。

**ArcaneDoorに関与する攻撃者によるCiscoファイアウォールへの持続的な侵害**
「UAT-4356」として追跡される攻撃者が、Cisco ASAおよびFTDの脆弱性を組み合わせて悪用し、正規の監視手法では検出が困難な「FIRESTARTER」バックドアなどのマルウェアを設置していたことが判明しました [6, 7]。米CISAの評価によれば、この侵害は2025年9月初旬には既に発生しており、パッチ適用前からのゼロデイ攻撃の可能性が指摘されています [7]。

## 🟠 脆弱性情報
**Cisco Catalyst SD-WAN Controller の認証バイパスの脆弱性（CVE-2026-20182）**
*   **CVSSスコア:** 10.0（最大深刻度）[1, 8]
*   **概要:** ピアリング認証メカニズムの不具合により、リモートの攻撃者が細工したリクエストを送信することで、認証をバイパスして管理権限を取得できる可能性があります [1, 8, 9]。
*   **推奨対応:** Ciscoが公開している最新のセキュリティ更新プログラムを速やかに適用すること、および「/var/log/auth.log」などのログに不審なIPアドレスからのアクセスがないか確認することが推奨されています [9, 10]。

**Google Chromeにおける複数の重大な脆弱性（CVE-2026-8509, CVE-2026-8510等）**
*   **CVSSスコア:** Critical（重大）[11, 12]
*   **概要:** Chrome 148のアップデートで修正された79件の脆弱性のうち14件が「重大」に分類されており、WebMLにおけるヒープバッファオーバーフローやSkiaにおける整数オーバーフローなどが含まれます [11]。
*   **推奨対応:** Windows/macOS向けにはバージョン148.0.7778.167/168以降、Linux向けには148.0.7778.167以降へ更新してください [13]。

**オンプレミス版 Microsoft Exchange Server のなりすましの脆弱性（CVE-2026-42897）**
*   **CVSSスコア:** 8.1 [14]
*   **概要:** クロスサイトスクリプティング（XSS）の欠陥に起因し、細工されたメールをユーザーがOutlook Web Accessで開くことで、ブラウザ上で任意のJavaScriptが実行され、なりすましが行われる可能性があります [14, 15]。
*   **推奨対応:** Microsoftは「Exchange Emergency Mitigation Service」を通じた自動緩和策を提供しており、それが利用できない場合は「Exchange on-premises Mitigation Tool (EOMT)」を用いた手動対応を求めています [16, 17]。

## 🟡 攻撃キャンペーン
**Cisco SD-WANを標的とした広範なマルウェア展開キャンペーン**
2026年3月以降、複数の攻撃クラスターがCisco SD-WANの脆弱性を連鎖的に悪用し、公表されている概念実証（PoC）コードを武器化して攻撃を展開しています [2, 18]。このキャンペーンでは、レッドチーミングフレームワーク「AdaptixC2」をベースにしたマルウェアや、C2フレームワーク「Sliver」、アセットマッピングツール「KScan」などが使用されています [4]。

**フィッシング経由のランサムウェア攻撃チェーン**
初期のフィッシング攻撃から始まり、最終的にランサムウェアの展開に至る一連の攻撃チェーンが確認されています [19, 20]。このチェーンを断ち切るために、信頼されている資産の悪用を防ぐ対策の重要性が強調されています [19, 20]。

## 🤖 AIセキュリティ
*   **G7諸国によるAI SBOMガイドライン:** G7諸国が、AIモデルやシステムの構成要素を管理するための「AI SBOM（Software Bill of Materials）」に関するガイダンスをリリースしました [21-23]。
*   **WebMLの脆弱性:** Google ChromeのWeb Machine Learning（WebML）コンポーネントにおいて、リモートコード実行につながる可能性がある「重大」な脆弱性（CVE-2026-8509）が修正されました [11]。
*   **AIデータセンターのセキュリティ:** AI特化型データセンターにおいて、パフォーマンスを損なわずにセキュリティを確保するため、CPUベースのエージェントから独立したDPU（Data Processing Unit）ベースのアーキテクチャへの移行が議論されています [24-26]。

## 🔵 業界動向
*   **セキュリティとパフォーマンスの両立:** 高パフォーマンス計算が求められる環境において、ホストCPUに負荷をかけないDPUベースのハードウェア分離型セキュリティが新たなトレンドとなっています [26-28]。
*   **官民パートナーシップの強化:** 国家のサイバーレジリエンスを確保するため、政府と民間セクターとのより迅速かつ深い連携が必要であるとの提言がなされています [29-31]。
*   **自律型エージェントへの対抗:** AIによる自動化された攻撃スピードに対抗するため、企業側も「エージェント型（Agentic）」のAI駆動防御を導入する必要性が指摘されています [32-34]。
*   **サードパーティリスク管理の課題:** 組織がサードパーティリスク管理に多額の投資を行っているにもかかわらず、侵害や盲点が依然として解消されていない実態が指摘されています [35-37]。

## 🇯🇵 国内情報
**JPCERT/CCによるマイクロソフト月例更新プログラム（2026年5月）の注意喚起**
2026年5月13日に公開されたマイクロソフト製品の脆弱性修正プログラムについて、リモートからの任意のコード実行の恐れがあるとして、速やかな適用を促しています [38, 39]。

**GUARDIANWALL MailSuite のゼロデイ攻撃に関する注意喚起**
キヤノンマーケティングジャパンの「GUARDIANWALL MailSuite（オンプレミス版）」において、認証なしで任意のコード実行が可能なバッファオーバーフローの脆弱性（CVE-2026-32661）が悪用されていることが公表されました [40]。JPCERT/CCはパッチの適用と侵害有無の調査を強く推奨しています [40, 41]。

**Cisco ASA/FTD脆弱性の国内での多数観測**
JPCERT/CCは、Cisco ASAおよびFTDの既知の脆弱性（CVE-2025-20333等）の影響を受けるホストが依然として国内で多数存在していることを確認し、改めて注意を呼びかけています [42]。
