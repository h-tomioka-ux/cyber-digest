# 🔵 Cisco ダイジェスト 2026-05-12

> ソース: RSS 3フィード (12記事) ／ NotebookLM分析

2026年5月12日に取得された、シスコの最新テクノロジー、セキュリティ、およびグローバルな取り組みに関する分析は以下の通りです。

## 🛡️ セキュリティプロダクト
*   **Cisco Secure Firewall 6100 シリーズ**
    AI規模のインフラ向けに設計された次世代ファイアウォールで、最大600Gbpsのスループットを実現しつつ、従来モデル比で設置スペースを最大80%、消費電力を60%削減します [1, 2]。専用ハードウェアによる暗号化オフロード機能を備え、プライバシーを保護しつつラインレートでの脅威識別が可能です [3]。
*   **Cisco Secure Access「Security Insights」**
    SASEプラットフォームに統合された新しい分析ダッシュボードで、リスクの高いユーザーやDLP（データ漏洩防止）違反、GenAIツールの利用状況などをランク付けして可視化します [4-6]。これにより、SOCアナリストは数百万のログを読み解くことなく、優先順位に基づいた調査を迅速に開始できます [7, 8]。
*   **DefenseClaw (オープンソースプロジェクト)**
    急成長するAIエージェントプラットフォーム「OpenClaw」向けのガバナンスレイヤーで、スキルのインストール前スキャンや実行ループ内でのリアルタイムな脅威検出を提供します [9, 10]。Splunkとシームレスに連携し、エージェントの全挙動を最初からオブザーバビリティ（観測可能性）の対象とすることが可能です [11]。
*   **ポスト量子暗号（PQC）ロードマップ**
    将来的な量子コンピュータによる攻撃に備え、Secure FirewallにML-KEMやML-DSAといったNIST標準アルゴリズムを導入する計画を策定しています [12-14]。FTD 10.5（2026年後半予定）より、VPNトンネルなどの耐量子保護が順次実装されます [13, 15]。

## 🔒 セキュリティアドバイザリ
*   **UAT-4356によるCisco Firepowerデバイスへの攻撃**
    *   **CVE番号**: CVE-2025-20333, CVE-2025-20362
    *   **概要**: 国家支援型攻撃グループUAT-4356（ArcaneDoor）が、Firepowerデバイスのn-day脆弱性を悪用して「FIRESTARTER」バックドアを送り込んでいることを確認しました [16]。このマルウェアはLINAプロセスに自身を挿入し、特定のXMLリクエストを通じて任意コードの実行を可能にします [17, 18]。
    *   **推奨対応**: シスコのセキュリティアドバイザリに従い、速やかに該当ソフトウェアをアップグレードすることを推奨します [17, 19]。また、デバイスの再イメージ化による軽減策や、Snortルール（65340, 46897等）の活用も有効です [19]。
*   **Luaベースのマルウェア「LucidRook」による標的型攻撃**
    *   **CVE番号**: 該当なし
    *   **概要**: 台湾のNGOや大学を標的に、Luaインタープリタを組み込んだ高度なステージャ「LucidRook」が配布されています [20]。攻撃者は正規のメールインフラや公開FTPサーバーを悪用し、ステルス性の高い多段階の感染チェーンを展開しています [21-23]。
    *   **推奨対応**: Talosが公開しているIOC（侵害指標）に基づくブロックや、ClamAVシグネチャ、SNORTルールの適用による検知体制の強化が必要です [24, 25]。
*   **OpenClawのリモートコード実行脆弱性**
    *   **CVE番号**: CVE-2026-25253
    *   **概要**: パーソナルAIオペレーティングシステム「OpenClaw」において、悪意のあるWebページにアクセスするだけでエージェントがハイジャックされる重大な脆弱性が報告されました [26]。

## 🔗 ネットワーク・インフラ
*   **次世代データセンター冷却ソリューション**: 102.4Tbpsのスループットを実現するCisco N9000/8000システムに、100%直接液冷（ダイレクト・リキッド・クーリング）デザインを導入し、エネルギー効率を約70%向上させました [27, 28]。
*   **CiscoソリューションのGoogle Cloud Marketplace提供**: Secure WorkloadやAI Defenseなどのセキュリティ製品がマーケットプレイス経由でパートナー主導で購入可能になり、クラウド支出との整合性が容易になりました [29, 30]。
*   **産業用ワイヤレス「Catalyst IW9167E」**: 鉱山などの過酷な環境向けに設計されたIP66/IP67定格の高耐久アクセスポイントで、URWB技術により移動体へのゼロパケットロス接続を実現します [31]。
*   **ハイブリッドメッシュファイアウォール**: クラウド、エッジ、オンプレミスに分散したセキュリティポリシーを一元管理し、ワークロードに近い場所で保護を適用するアーキテクチャを展開しています [32, 33]。
*   **CCIE Collaboration ラボアップデート**: 2026年5月より試験環境をv15へアップグレードし、IM & Presenceの廃止やWebex Calling統合、AIを活用した学習支援など、クラウドネイティブな環境への移行を反映させています [34-37]。

## 🤖 AI・自動化
*   **Cisco AI Defense と Google ADK の統合**
    GoogleのAgent Development Kit (ADK)で構築されたAIエージェントに対し、ランタイム保護を直接組み込むことができます [38, 39]。開発者は数行のコードを追加するだけで、ローカル開発からクラウド展開まで一貫したセキュリティポリシーを適用可能です [40, 41]。
*   **Cisco LLM Security Leaderboard**
    大規模言語モデル（LLM）の敵対的攻撃に対する耐性を客観的に評価する公開リソースで、ベースモデル固有のセキュリティ態勢をランク付けします [42-44]。
*   **モデル・プロブナンス憲章 (Model Provenance Constitution)**
    AIサプライチェーンの安全性を確保するため、モデルの重み（ウェイト）レベルでの導出関係を厳格に定義し、脆弱性の継承やライセンス義務を追跡可能にするフレームワークです [45-47]。
*   **意図ベースのネットワーク自動化**
    RedditでのAMAによれば、シスコはAIを自律的な存在ではなく「推論レイヤー」として位置づけ、人間が最終承認を行う仕組み（human-in-the-loop）による自動化を推進しています [48, 49]。
