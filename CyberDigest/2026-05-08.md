# 🛡️ サイバーセキュリティ・ダイジェスト 2026-05-08

> ソース: RSS 6フィード (12記事) ／ NotebookLM分析

2026年5月8日に取得された最新のサイバーセキュリティニュースに基づき、以下の通り分析結果を報告します。

## 🔴 重大インシデント
*   **中国の国家支援が疑われるPalo Alto Networks製品へのゼロデイ攻撃**
    Palo Alto Networksは、自社製品のUser-ID認証ポータルに存在するゼロデイ脆弱性（**CVE-2026-0300**）が悪用されたことを公表しました [1, 2]。CL-STA-1132と呼ばれる「国家支援の可能性が高い」攻撃グループが、root権限でのリモートコード実行やActive Directoryの列挙を行っており、その手法は中国のAPTグループ（Volt TyphoonやAPT41）の典型的な特徴と一致しています [3-5]。
*   **Cisco製ファイアウォールにおける持続的なバックドア攻撃**
    Cisco ASAおよびFTD製品において、脆弱性を修正した後も機器内に留まり続ける持続化メカニズム（**FIRESTARTERバックドア**）が確認されました [6, 7]。この攻撃は2025年9月頃から発生しており、一般的な監視・検知手法では発見が困難な高度なステルス性を備えていることが指摘されています [7]。

## 🟠 脆弱性情報
*   **Ivanti Endpoint Manager Mobile (EPMM) のRCE脆弱性**
    *   **CVE番号**: CVE-2026-6973 [8]
    *   **CVSSスコア**: **7.2** [8]
    *   **概要**: 管理者権限を持つ認証済みユーザーが、不適切な入力検証を悪用してリモートからコードを実行できる脆弱性です。野生下での限定的な悪用が確認されており、米CISAの「悪用が確認済みの脆弱性（KEV）」カタログに追加されました [8-10]。
    *   **推奨対応**: 速やかに修正バージョン（12.6.1.1, 12.7.0.1, 12.8.0.1以降）へアップデートし、必要に応じて認証情報のローテーションを実施してください [8, 9]。
*   **Microsoft SharePoint Server のなりすまし脆弱性**
    *   **CVE番号**: CVE-2026-32201 [11]
    *   **CVSSスコア**: 記載なし
    *   **概要**: 認証不要でネットワーク経由でのなりすましが行われる可能性がある脆弱性で、既に悪用が公表されています [11]。
    *   **推奨対応**: 2026年4月のセキュリティ更新プログラムを適用してください [11, 12]。
*   **Adobe AcrobatおよびReaderの脆弱性**
    *   **識別番号**: APSB26-44 [13]
    *   **CVSSスコア**: 記載なし
    *   **概要**: 特製されたコンテンツを開くことで、任意のコードが実行されるなどの恐れがあります [13]。
    *   **推奨対応**: ヘルプメニューからアップデートをチェックし、最新バージョンへ更新してください [14]。

## 🟡 攻撃キャンペーン
*   **クラウドインフラを標的とする「PCPJack」認証情報窃取キャンペーン**
    AWS、Azure、Kubernetes、Dockerなどのクラウドサービスを標的に、ワームのように拡散する新たなフレームワーク「**PCPJack**」が発見されました [15, 16]。このツールは5つの脆弱性を悪用して横展開を行い、盗み出した認証情報をTelegram経由で攻撃者に送信するほか、競合する攻撃グループ（TeamPCP）の痕跡を排除する機能を備えています [15, 17, 18]。
*   **「Patient Zero（第0号患者）」を狙うAI悪用フィッシング**
    攻撃者が生成AIを利用して、従来のフィルタリングを回避する高度にパーソナライズされたフィッシングメールを作成し、社内への最初の足がかり（Patient Zero）を作るキャンペーンが増加しています [19, 20]。

## 🤖 AIセキュリティ
*   **開発者ツールにおけるプロンプトインジェクションの脅威**
    Claude Code、Gemini CLI、GitHub Copilot AgentsなどのAIツールが、コメントを介した**プロンプトインジェクション**に対して脆弱であることが報告されています [21]。
*   **AIによる攻撃の高速化と防御側へのAI導入**
    攻撃者がAIを利用してリモートアクセスの侵害速度を上げている現状に対抗するため、企業はAI駆動の自律型エージェント（agentic defense）を導入し、AIにはAIで立ち向かう必要があると提唱されています [22, 23]。
*   **AIの安全性を巡る法的・倫理的議論**
    イーロン・マスク氏とOpenAIのリーダー間の裁判において、汎用人工知能（AGI）が人類にもたらすリスクや安全性への懸念が主要な焦点となっています [24-26]。

## 🔵 業界動向

*   **セキュリティ企業のM&A**: Boost Securityが、コードの到達可能性分析を行うSecureIQxと、AIコードレビュープラットフォームのKorbit.aiの2社を買収しました [29]。
*   **米政府による監視強化**: CISAはIvantiの脆弱性をKEVカタログに追加し、連邦政府機関に対して2026年5月10日までの対応を義務付けました [10]。
*   **官民連携の重要性**: 国家の強靭性を確保するためには、民間セクターとのより迅速かつ深いパートナーシップが不可欠であるとの見解が示されています [30]。

## 🇯🇵 国内情報
*   **JPCERT/CCによる注意喚起の継続**
    JPCERT/CCは、2026年4月のマイクロソフト製品の月例更新 [11]、Adobe Acrobat/Reader [31]、およびCisco ASA/FTD [32] に関する注意喚起を更新または公開しています。
*   **国内におけるCisco製品への侵害リスク**
    JPCERT/CCの調査により、攻撃の対象となるCisco ASA/FTDの脆弱なホストが**日本国内に多数存在している**ことが確認されています [33]。修正パッチ適用前に侵害されていた場合、バックドアが潜伏し続けている可能性があるため、侵害調査の実施が推奨されています [7, 33]。
